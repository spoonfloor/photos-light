============================================================================
TARGET BEHAVIOR (per flow)
============================================================================

For each flow fatal exit:

  1. Stop inflight UI (spinners, tickers, EventSource/SSE readers)
  2. Abort controller if still active (skip user confirm on error)
  3. Show concise error in overlay OR toast (overlay preferred if already open)
  4. Present exactly one primary exit: "Close" / "Done" → dismissOverlay OR
     cancelRecovery(skipConfirm: true) with correct reloadGrid
  5. syncOverlayPhase(flowKey, null) — lifecycle returns to idle
  6. restoreLibraryShellAfterFlowDismiss when overlay hid and grid/shell may be blank
  7. reloadGrid when partial work may have changed catalog (policy per flow)

reloadGrid policy (defaults — don't block on user):
  Add:    true if importedCount > 0, else false
  Clean:  true (checkpoint may have saved progress)
  Convert: false unless library switch completed (usually false on failure)
